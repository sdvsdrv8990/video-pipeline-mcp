# История теста: cache_injection

## Статус: РАБОЧИЙ ТЕСТ

## Назначение
Проверка что файрвол отклоняет попытки отравления кеша сервера.

## Тип теста
Security / Integration (тест защиты сервера)

## Что тестируем
1. Инъекция через file_upload
2. XSS в payload
3. Prototype pollution попытка
4. Изоляция кеша от данных

## Зачем нужен
1. **Безопасность:** Отравление кеша может сломать данные
2. **Изоляция:** Workspace должен быть изолирован
3. **Верификация:** Данные не повреждаются

## Что хотим увидеть
- Инъекция отклоняется файрволом
- Кеш остаётся целостным
- Workspace не повреждается

## Как отражает реальное поведение
Эмулирует попытку загрузить вредоносные данные через инструменты.

## Workspace
НЕТ — это тест защиты. Workspace не используется.

# История теста: bot_army

## Статус: РАБОЧИЙ ТЕСТ

## Назначение
Проверка что файрвол корректно реагирует на массовые подключения с подозрительным поведением.

## Тип теста
Security / Integration (тест защиты сервера)

## Что тестируем
1. Массовое подключение (много запросов с одного IP)
2. Поведенческий фактор (одинаковые payload'ы)
3. Бан по признакам (порог нарушений)
4. Восстановление после бана

## Зачем нужен
1. **Безопасность:** Армия ботов — реальная угроза
2. **Регрессия:** При изменении правил не сломать защиту
3. **Верификация:** Убедиться что бан срабатывает

## Что хотим увидеть
- Rate limiting срабатывает после N запросов
- Бан наступает после ban_after_violations
- Легитимные запросы проходят после бана
- Логируется попытка атаки

## Как отражает реальное поведение
Эмулирует подключение множества ботов к MCP-серверу. Каждый бот отправляет
одинаковые payload'ы — типичный паттерн автоматизированной атаки.

## Критерии успеха
- Rate limiting работает (после max_requests → rate_limit)
- Бан наступает (после ban_after → block)
- Легитимные запросы не затронуты

## Workspace
НЕТ — это тест защиты, не инструментов. Workspace не используется.

## Регрессии
- Если пороги изменятся → бан может не сработать или сработать рано
- Если логика аномалий изменится → поведенческий фактор не поймает

# История теста: firewall

## Статус: РАБОЧИЙ ТЕСТ (не заглушка)

## Назначение
Проверка что файрвол корректно фильтрует атаки и пропускает легитимные запросы.

## Тип теста
Security / Integration (постоянная регрессия)

## Что тестируем
1. Блокировка prompt injection
2. Rate limiting
3. IP blocking
4. Прохождение легитимных запросов

## Зачем нужен
1. **Безопасность:** Файрвол — первая линия защиты
2. **Регрессия:** При изменении правил не сломать защиту
3. **Верификация:** Убедиться что защита работает

## Что хотим увидеть
- Prompt injection блокируется
- Rate limiting срабатывает при превышении лимита
- IP blocking работает (блокировка/разблокировка)
- Легитимные запросы проходят без задержек

## Критерии успеха
- Все 4 теста пройдены
- Нет false positives на легитимных запросах
- Время проверки < 100ms

## Регрессии
- Если правила ужесточатся → ложные блокировки
- Если правила ослабнут → атаки пройдут

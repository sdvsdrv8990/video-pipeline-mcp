# История теста: config_change

## Статус: РАБОЧИЙ ТЕСТ

## Назначение
Проверка адаптации сервера к изменению конфигурации оборудования.

## Тип теста
System / Integration (тест поведения сервера)

## Что тестируем
1. Изменение CPU/RAM/DISK
2. Адаптация параметров
3. Уведомление Claude
4. Продолжение работы

## Зачем нужен
1. **Адаптивность:** Сервер должен подстраиваться
2. **Уведомления:** Claude должен знать об изменениях
3. **Доступность:** Работа продолжается

## Что хотим увидеть
- Сервер адаптируется к новым ресурсам
- Claude получает уведомление
- Автоматическая оптимизация параметров

## Как отражает реальное поведение
Эмулирует изменение конфигурации сервера в процессе работы.

## Workspace
НЕТ — это тест поведения сервера. Workspace не используется.

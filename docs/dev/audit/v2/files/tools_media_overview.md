# Q&A: Media-инструменты — обзор

> **Роль:** тонкие обёртки поверх core/providers/. Три способности: STT/TTS/IMG.
> **Сквозное:** [G4](../global.md#g4-три-фазы-инструмента-trigger--poll--download), [G5](../global.md#g5-философия-ошибок-ловим-и-кодируем-не-чиним-отдаём-полный-текст), [G16](../global.md#g16-незавершённость-должна-кричать-честная-заглушка-а-не-молчать).
> **Статус:** заглушки с `NotImplementedError` (честная незавершённость).
> **Навигация:** `tools/media/` → `core/providers/{stt,tts,img}/`.

## Три способности
| Способность | Адаптер | Бэкенд |
|---|---|---|
| Транскрибация (STT) | core/providers/stt/ | stable-ts, ЛОКАЛЬНО |
| Озвучка (TTS) | core/providers/tts/ | LiteLLM (ОБЛАКО/локально) |
| Картинки (IMG) | core/providers/img/ | LiteLLM (ОБЛАКО/локально) |

## Паттерн вызова
**Sync** (короткий TTS, локальный STT): `trigger_*` → сразу результат, `poll_*` вырождается.
**Async** (картинки FLUX/SD): `trigger_*` → `{task_id}`, `poll_*` опрос, `download_*` скачивание.
**Верификация обязательна всегда:** мало «провайдер сказал готово» — проверь файл на диске.

## Три природы сбоя
| Сбой | Код | Кто чинит |
|---|---|---|
| Локальный инференс | `LOCAL_INFERENCE_FAILED` | среда (модель↓, потом человек) |
| Внешний технический | `PROVIDER_FAILED` | сервер (retry + авто-цепочка) |
| Внешний контент-отказ | `CONTENT_REJECTED` | Claude (переформулировать/сменить) |

## Как результат попадает в данные
- Скачанный файл → папка ассетов видео
- Путь + ID → в таблицу через очередь
- Таймкоды из STT → листы SCENES и TRANSCRIPT

# Q&A: стратегия тестирования

> **Роль:** философия тестирования, быстрые/постоянные тесты, 5 вопросов, типы тестов безопасности.
> **Сквозное:** [G8](../global.md#g8-дисциплина-документирования-решений-5-вопросов--4-уровня).
> **Статус:** философия описана, тесты — заглушки (активируются после core/).
> **Навигация:** `tests/quick/` (быстрые) → `tests/<name>/` (постоянные) → `tests/config/scenarios.yaml`.

## Решение 1: Тест = сценарий поведения, не проверка кода
**Q:** что такое тест в нашем проекте?
**A:** сценарий реальных действий. Пользователь ставит задачу → эмулируем поведение Claude → проверяем результат. Не проверяем внутренности модулей.
**Связь:** SESSIONS.md §Приложение А, [G8](../global.md#g8-дисциплина-документирования-решений-5-вопросов--4-уровня).

## Решение 2: 5 вопросов перед тестом
**Q:** что ответить перед созданием теста?
**A:**
1. **Что тестируем?** — конкретный сценарий
2. **Зачем нужен этот тест?** — какую проблему решает
3. **Что хотим увидеть?** — ожидаемый результат
4. **Как тест отражает реальное поведение?** — эмуляция реальных действий
5. **Это постоянный тест или быстрый?** — определяет структуру
**Связь:** SESSIONS.md §Приложение А, workflow.md.

## Решение 3: Быстрые vs постоянные тесты
**Q:** чем отличаются?
**A:**
| Тип | Жизнь | Структура | Когда удалять |
|---|---|---|---|
| Быстрый | `tests/quick/` | `test_<что>.py` | После проверки |
| Постоянный | `tests/<name>/` | `test_<name>.py` + `history.md` | Если устарел |
**Цикл быстрого:** создали → проверили → удалили. Работаем.
**Цикл постоянного:** нашли баг/сценарий → создали тест → живёт → при изменениях проверяем.
**Связь:** SESSIONS.md §Приложение А.

## Решение 4: Типы тестов безопасности
**Q:** какие атаки эмулируем?
**A:**
| Тип | Что эмулируем | Что проверяем |
|---|---|---|
| Prompt injection | Вредоносные JSON-запросы | Блокировка инъекций |
| Cache overflow | Переполнение кеша | Сервер не падает |
| Bot army | 100+ одновременных запросов | Бан/детекция |
| Virus injection | Внедрение кода | Изоляция |
| Cache injection | Инъекция в кеш | Целостность данных |
| Slowloris | Медленные соединения | Ресурсы |
**Источники:** GitHub (mcp-watch, agentshield), OWASP LLM Top 10, CVE базы, наши bypass'ы.
**Цикл:** нашли bypass → создали тест → проверили → исправили → регрессия.
**Связь:** SESSIONS.md §Приложение А.

## Решение 5: Метрики
**Q:** что измерять?
**A:**
- **Ban rate:** % ботов, которых банит firewall
- **False positives:** % легитимных под баном
- **Response time:** время реакции на атаку
**Связь:** SESSIONS.md §Приложение А.

## Решение 6: Таблица планирования (для постоянных тестов)
**Q:** что заполнить перед созданием постоянного теста?
**A:**
| Тест | Что эмулируем | Ожидаемый результат |
|---|---|---|
| `firewall` | Prompt injection, rate limiting | Сервер блокирует атаки |
| `tool_discovery` | Запрос tools/list | Claude видит инструменты |
| `provider_errors` | Ошибки провайдеров | Маппинг на наши коды |
**Связь:** SESSIONS.md §Приложение А.

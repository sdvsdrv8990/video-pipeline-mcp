# Q&A: Видеомонтаж — сборка сцены

> **Роль:** монтаж компонует сцену из готовых исходников (не генерит их).
> **Сквозное:** [G4](../global.md#g4-три-фазы-инструмента-trigger--poll--download), [G11](../global.md#g11-поставщик-туннеля--cloudflare-tunnel-cloudflared).
> **Статус:** архитектура описана; FFmpeg как внешний MCP через адаптер.
> **Навигация:** `tools/video/` → `core/providers/ffmpeg/`.

## Поток данных
```
media (img_*/tts_*)  →  ИСХОДНИКИ (фон, персонаж, компоненты, музыка, звук)
   ↓
МОНТАЖ               →  компоновка: слои + позиция + цветокор + аудио + переходы
   ↓
готовая сцена → видео
```

## Движок: FFmpeg
FFmpeg выбран: headless, скриптуемый, детерминированный. DaVinci не подходит — GUI-приложение, не для сервера.
**Архитектура:** `core/providers/ffmpeg/wrappers/` + `ffmpeg_adapter.py` (единый интерфейс).

## Позиционирование: слоты поверх пикселей
- **База:** всё в пикселях (слот = пиксельный прямоугольник)
- **Сверху:** сетка/слоты (центр / низ / справа)
- Слот = именованный пиксельный прямоугольник, переопределяемый вручную.

## Два режима
- **Создание раскладки** (дорого, редко): слотов нет → задать с нуля
- **Применение раскладки** (дёшево, часто): слоты есть → ИИ назначает фрагменты

## Растягивание = вариация
Пиксельная правка размера/позиции — новая вариация мастера. Попадает в `ASSETS_USED` и `scene_uniqueness`.

## Где что лежит
| Артефакт | Вселенная | Место |
|---|---|---|
| Обёртки FFmpeg + адаптер | КОД (git) | `core/providers/ffmpeg/` |
| Раскладки сцен | ДАННЫЕ канала | `workspace/.../scene_layouts/` |
| Параметры рендера | ДАННЫЕ канала | `channel_config.RENDER_CONFIG` |

## Вывод рендера
Стадии: черновой → финальный. Место: `videos/<video>/renders/`. Имя: `{video_slug}_{render_stage}.mp4`. Verify обязателен.

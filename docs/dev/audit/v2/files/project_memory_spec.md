# Q&A: память проекта

> **Роль:** формат project_memory.md: что решили и что из этого вышло (не числа/статусы).
> **Сквозное:** [G9](../global.md#g9-две-вселенные-код-сервера-vs-workspace-данные).
> **Статус кода:** спека готова, реализация через fs_* инструменты.
> **Навигация:** `workspace/{channel|video}/project_memory.md` ↔ `table_set` (флаги) ↔ `_SESSION_LOG` (события).

## Решение 1: Три хранилища — разные вопросы
**Q:** чем память отличается от лога и таблиц?
**A:**
| Хранилище | Вопрос | Кто читает |
|---|---|---|
| `_SESSION_LOG` | что ПРОИЗОШЛО | машина |
| Таблицы (.xlsx) | какое СОСТОЯНИЕ и МЕТРИКИ | сервер/Claude |
| `project_memory.md` | что РЕШИЛИ и ЧТО ИЗ ЭТОГО ВЫШЛО | человек/Claude |
**Железное правило:** не дублировать таблицы. Число/статус → в таблице, в память НЕ копируется. Причина решения → в памяти, в таблице такого нет.
**Связь:** SESSIONS.md §Сессия 2 (state).

## Решение 2: Два уровня
**Q:** где живёт память?
**A:** на двух уровнях:
| Уровень | Файл | Что хранит |
|---|---|---|
| Видео | `…/videos/{video}/project_memory.md` | решения по ЭТОМУ видео |
| Канал | `…/channels/{channel}/project_memory.md` | кросс-видеовые решения |
**Связь:** SESSIONS.md §Сессия 2, [G9](../global.md#g9-две-вселенные-код-сервера-vs-workspace-данные).

## Решение 3: Замкнутая петля: сигнал → решение → флаг → результат
**Q:** как память связана с данными?
**A:**
```
1. скрипт-сверка поймал сигнал → факт в таблице (UNIQUENESS_ALERTS)
2. решение человека → ЗАПИСЬ в память: что решили + ПОЧЕМУ
3. Claude ставит флаг → signal_status + flag_source=human_decision
4. позже: РЕЗУЛЬТАТ дописывается в память → ссылка на эффект по ID
5. петля замкнута
```
**Связь:** SESSIONS.md §Сессия 2.

## Решение 4: Формат записи
**Q:** как выглядит запись в памяти?
**A:**
```markdown
## [дата] Решение: <короткий заголовок>
- **Контекст:** что произошло (ссылка на сигнал/ID)
- **Кто решил:** человек / Claude (с подтверждением)
- **Решение:** что именно сделали
- **Почему:** причина
- **Результат:** [дописывается позже] эффект по фактам
```
**Чего не должно быть:** числа/статусы из таблиц, сырые события (это _SESSION_LOG), флаги как таковые (они в таблице), догадки Claude без пометки.
**Связь:** SESSIONS.md §Сессия 2.

## Решение 5: Как пишется
**Q:** кто записывает память?
**A:** память — файл, не таблица: пишется `fs_*` инструментами. Флаг сигнала живёт в таблице (`signal_status`, `flag_source`) и меняется `table_set`.
**Связь:** SESSIONS.md §Сессия 2.

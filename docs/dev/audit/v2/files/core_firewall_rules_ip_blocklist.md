# Q&A: core/firewall/rules/ip_blocklist.py

> **Роль:** хранение/проверка заблокированных IP с TTL; шаг 1 в `Firewall.check()` (`is_blocked`).
> **Сквозное:** [G3](../global.md#g3-firewall-перед-ядром); [G12](../global.md#g12-эфемерное-in-process-состояние-файрвола) (`_blocked` в памяти, ключ по IP × один IP туннеля — корень D14).
> **Статус кода:** реализован; корректен по механике, но опасен в связке с туннелем (D14) и `ban_duration_hours=0`.
> **Навигация (знать не читая):** `core/firewall/rules/ip_blocklist.py`. Поверхность: `is_blocked(ip)->bool`, `block(ip,reason)`, `unblock(ip)`, `get_blocked_count()`, `get_blocked_list()`. Вызывается: `is_blocked` — `firewall.py` шаг 1; `block` — там же после `should_ban` и из `Firewall.block_ip`. Состояние: `_blocked: dict[ip→BlockedIP]`.
> **Аудит-линзы:** security-reviewer (осн.), test-master. Доказано запуском на `.venv`.

## Решение 1: бан с TTL и авто-истечением
**Q:** бан навсегда или временный?
**A:** `expires_at = now + ban_duration_hours*3600`; `is_blocked` снимает истёкший бан на лету; `None` = навсегда.
**Alt:** только вечный бан — отброшено: копит мусор, наказывает динамические IP.
**Поток данных:** `is_blocked(ip) → есть? → expires_at и now>expires? → del+False : True`.
**Регрессия / где проявится:** `ban_duration_hours=0` (или None) → `expires_at=None` → **вечный бан**. **Доказано:** `IPBlocklist(ban_duration_hours=0).block(...)` → `expires_at=None`. В связке с туннелем (один IP, [D14](../AUDIT.md#-d14)) вечный бvan общего IP = Claude отрезан навсегда до рестарта. Значение `0` интуитивно читается как «без длительности», а даёт максимально жёсткий эффект — footgun.

## Решение 2: ленивая + пакетная чистка истёкших
**Q:** когда удалять истёкшие баны?
**A:** лениво в `is_blocked` (по обращению) и пакетно `_cleanup_expired` в `get_blocked_count/list`.
**Alt:** фоновый таймер — отброшено: лишняя инфраструктура ради словаря.
**Регрессия:** без обращения истёкшие висят до следующего подсчёта (незначительно; память).

## Решение 3: `BlockedIP` хранит причину и времена
**Q:** что помнить о бане, кроме факта?
**A:** `reason`, `blocked_at`, `expires_at` — для аудита и `get_blocked_list`.
**Alt:** `set` из IP — отброшено: теряется причина/срок, бан не объяснить.
**Регрессия:** — .

## Открытые вопросы файла
- **🟡 D14 (подтверждён запуском):** ключ бана — IP; за cloudflared весь Claude = один IP. **Доказано:** `block(tunnel_ip)` → `is_blocked(tunnel_ip)=True` = отрезан весь трафик, не один клиент. См. [../AUDIT.md#-d14](../AUDIT.md#-d14).
- **Связка с [D16](../AUDIT.md#-d16):** `unblock` не сбрасывает violations в rate_limiter → разбан общего IP бесполезен, следующий всплеск ре-банит.
- **Нет persistent allowlist** (напр. IP туннеля Claude) — простейшая защита от D14/D16 отсутствует.
- **Сквозное (→ global.md):** состояние в памяти — рестарт снимает все баны, воркеры не делят блоклист. Для прода — общий стор (Redis/файл с локом). Кандидат в `G#` вместе с rate_limiter.

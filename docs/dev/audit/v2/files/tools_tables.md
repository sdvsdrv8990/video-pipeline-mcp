# Q&A: табличные инструменты (Table Data Layer)

> **Роль:** работа с табличными данными. Generic-движок; различия только в схемах.
> **Сквозное:** [G2](../global.md#g2-единый-конверт-ответа-toolresult), [G9](../global.md#g9-две-вселенные-код-сервера-vs-workspace-данные).
> **Статус:** реализованы; 5 примитивов + json_* очередь.
> **Навигация:** `tools/tables/` + `config/ops/tables.ops.yaml` → `core/state/`.

## Ключевой закон
**Поведение задаётся конфигом (схемой таблицы), а не Python-кодом.** В коде нет `if table == "video_data"`.

## Модель данных
### Единица хранения — строка по ID (не столбцы)
Причина: прерванная запись при столбцах → «разъезжаются» индексы. Привязка к ID убирает этот класс ошибок.

### Столбец — проекция поверх строк
`get_column(table, "status")` → `{ "VID_fin_a1b2": "DRAFT" }`

### Запись — read-modify-write по строке
Даже для одного поля: взять строку → поменять → положить обратно.

## Пять примитивов
| Примитив | Назначение |
|---|---|
| `table_get_column` | проекция столбца `{id: value}` |
| `table_get_row` | одна строка `{column: value}` |
| `table_set` | изменить поле (RMW) |
| `table_append` | новая строка (ID от сервера) |
| `table_delete` | удалить строку |

## Очередь записи
Одна последовательная очередь на каждую таблицу. Чтения параллельны и не блокируются.

## Защита формул
Запись в `computed: true` → немедленный отказ. Формулы не перезаписываются.

## Стыковка с json_*
- `json_read_snapshot` — отдаёт `read.json`
- `json_push_to_queue` — кладёт намерение в `pending_updates`
- `json_execute_queue` — применяет очередь через RMW
- `json_clear_queue` — очистка

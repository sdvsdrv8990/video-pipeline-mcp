# Q&A: Excel Engine — структура таблиц

> **Роль:** полный CRUD над структурой (скелетом) Excel-книг. 14 инструментов.
> **Сквозное:** [G2](../global.md#g2-единый-конверт-ответа-toolresult), [G9](../global.md#g9-две-вселенные-код-сервера-vs-workspace-данные).
> **Статус:** реализованы.
> **Навигация:** `tools/excel_engine/` + `config/ops/excel.ops.yaml`.

## Инструменты
| Инструмент | Назначение |
|---|---|
| `excel_create_workbook` | новый .xlsx |
| `excel_add_sheet` / `rename_sheet` / `delete_sheet` / `reorder_sheets` | листы |
| `excel_add_column` / `delete_column` / `move_column` | столбцы |
| `excel_insert_formula` | формула в ячейку/диапазон |
| `excel_apply_formatting` | стили + условное форматирование |
| `excel_set_validation` | выпадающий список (Data Validation) |
| `excel_read_range` | сырой диапазон — ТОЛЬКО отладка |
| `excel_validate_formulas` | поиск `#REF!`/`#VALUE!` |

## Граница: структура vs данные
| | excel_* | json_* + примитивы |
|---|---|---|
| Что меняет | **форму** (листы, столбцы, формулы) | **значения** (строки, ячейки) |
| Как часто | редко | постоянно |
| Путь записи | напрямую | только через очередь |

**`excel_*` меняет форму таблицы, очередь меняет её значения.**

## Жёсткие границы
- `excel_*` меняет ТОЛЬКО структуру. Данные — через очередь.
- `excel_read_range` — отладочный, НЕ рабочий путь. Чтение — через `json_read_snapshot`.
- Защита формул: `insert_formula` не перезаписывает критические формулы молча.
- Синхронизация: после ручной правки → перегенерация `schema.json`.

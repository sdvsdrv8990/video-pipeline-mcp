# Q&A: пайплайн — бизнес-логика процессов

> **Роль:** 4 точки входа = наборы предусловий, единая библиотека шагов.
> **Сквозное:** [G4](../global.md#g4-три-фазы-инструмента-trigger--poll--download), [G1](../global.md#g1-роль-сервера-исолнительвалидатор-оркестратор--снаружи).
> **Статус кода:** заглушки (точки входа и шаги запланированы).
> **Навигация:** `pipeline/entry_points/` → `pipeline/steps/` → `tools/*`.

## Решение 1: 4 точки входа
**Q:** сколько воркфлоу и чем отличаются?
**A:** 4 точки входа = наборы предусловий, НЕ 4 копии воркфлоу:
| Вход | Предусловие |
|---|---|
| `cold_start.py` | ничего нет, начинаем с нуля |
| `project_ready.py` | структура создана, данные загружены |
| `competitor_ready.py` | конкуренты проанализированы |
| `assets_only.py` | только генерация ассетов |
**Принцип:** точки входа проверяют, какие данные уже есть, и роняют Claude в нужную точку общей библиотеки шагов.
**Связь:** SESSIONS.md §Сессия 5.

## Решение 2: Единая библиотека шагов
**Q:** как переиспользовать логику между точками входа?
**A:** `pipeline/steps/` — производственные шаги, описанные ОДИН раз:
| Шаг | Назначение |
|---|---|
| `write_script.step.py` | написание сценария |
| `tts_audio.step.py` | озвучка |
| `transcription.step.py` | транскрибация |
**Принцип:** шаги переиспользуются всеми входами. Производственные шаги не зависят от точки входа.
**Связь:** SESSIONS.md §Сессия 5.

## Решение 3: Пайплайн = сервер, не Claude
**Q:** кто оркестрирует пайплайн?
**A:** Claude. Сервер предоставляет точки входа и шаги как инструменты. Claude читает `read.json`, понимает состояние, выбирает шаг, шлёт запрос. Сервер исполняет.
**Связь:** [G1](../global.md#g1-роль-сервера-исолнительвалидатор-оркестратор--снаружи).

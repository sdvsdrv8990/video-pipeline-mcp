# Q&A: core/firewall/rules/injection_detector.py

> **Роль:** поиск prompt-injection паттернов (блоклист подстрок) в параметрах запроса; шаг 3 в `Firewall.check()`.
> **Сквозное:** [G3](../global.md#g3-firewall-перед-ядром) (firewall перед ядром).
> **Статус кода:** реализован; концептуально слабый (FP/FN, D7) + баг разделяемого списка (D15).
> **Навигация (знать не читая):** `core/firewall/rules/injection_detector.py`. Публичная поверхность: `detect(params)->bool`, `get_attempts()->int`, `add_pattern`/`remove_pattern`. Вызывается из `core/firewall/firewall.py` шаг 3. Сигнатуры — `DEFAULT_PATTERNS` (модульный список, 21 паттерн) → класс `InjectionDetector` → `detect` → приватные `_scan_value`/`_check_string`.
> **Аудит-линзы:** security-reviewer (осн.), mcp-developer (контракт/проводка), test-master (регрессии). Находки доказаны запуском на `.venv`.

## Решение 1: рекурсивное сканирование всех строковых значений
**Q:** где искать инъекцию в произвольном `params`?
**A:** `_scan_value` рекурсивно обходит `dict`/`list`/`tuple`/`str` — паттерн может быть в любой вложенности (indirect injection через вложенные поля).
**Alt:** проверять только верхний уровень — отброшено: инъекция прячется во вложенных полях.
**Поток данных:** `params → _scan_value → _check_string(text.lower())`; первый матч короткозамыкает (`any(...)`).
**Регрессия / где проявится:** глубокая рекурсия по большому/глубокому `params` — стоимость на горячем пути (каждый запрос). При гигантском вложенном `params` — заметная задержка перед ядром; DoS-амплификация (много вложений × 21 паттерн).

## Решение 2: блоклист подстрок, регистронезависимо
**Q:** как задавать сигнатуры?
**A:** список подстрок (`DEFAULT_PATTERNS`), сравнение по `lower()`, `in` (подстрока без границ слова).
**Alt:** regex/семантика — мощнее, но дороже/сложнее; выбрали простые подстроки.
**Регрессия / дефект → [D7](../AUDIT.md#-d7):** высокий FP + нулевая стойкость. **Доказано запуском:**
```
detect("act as a narrator for this scene")           -> True   # FP: легитимная озвучка
detect("disregard the previous take, re-render")     -> True   # FP: штатный пере-рендер
detect("i g n o r e   p r e v i o u s")               -> False  # FN: пробелы обходят
detect(base64("ignore previous"))                     -> False  # FN: threat_landscape §2.1
detect("ｉgnore previous")  (unicode)                  -> False  # FN: гомоглиф обходит
```
Примечание: v1 приводил пример `"override"⊂"overridden"` — эмпирически **неверно** (→False); реальные FP идут через `"act as"`/`"disregard"`.
**Где проявится в системе:** легитимные инструкции пайплайна (нарратор, «disregard previous take») получают `BLOCK`/`SECURITY_VIOLATION` вместо исполнения — пользователь видит «отказ безопасности» на нормальной операции. Одновременно реальная атака в base64/unicode проходит насквозь.

## Решение 3: дефолт в коде + переопределение из конфига + мутаторы
**Q:** откуда брать список и как расширять?
**A:** `patterns or DEFAULT_PATTERNS`; `add_pattern`/`remove_pattern` меняют `self.patterns`.
**Alt:** только конфиг (риск пустого списка = защита выключена) / только код (негибко) — отброшены.
**Регрессия / новый дефект → [D15](../AUDIT.md#-d15):** `self.patterns = patterns or DEFAULT_PATTERNS` присваивает **тот же** объект-список без копии. `add_pattern` на инстансе с дефолтами мутирует **модульный** `DEFAULT_PATTERNS`. **Доказано:** `add_pattern("zzz")` → `'zzz' in DEFAULT_PATTERNS == True`. **Где проявится:** правки паттернов одного firewall-инстанса протекают на все другие и живут весь процесс; в тестах — заражение состояния между кейсами (нарушение изоляции). **Фикс:** `self.patterns = list(patterns) if patterns is not None else list(DEFAULT_PATTERNS)`.

## Открытые вопросы файла
- **🟡 D7 (концепция + FP/FN):** в модели MCP оркестратор (Claude) — доверенная сторона; ловить инъекцию в им же сформированных `params` — борьба не с тем противником. Детерминированная защита важнее: auth транспорта ([D3](../AUDIT.md#-d3)), allowlist инструментов, песочница ФС ([D1](../AUDIT.md#-d1)), отсутствие `eval`. Ориентир: OWASP LLM01; для семантики — `protectai/llm-guard` (вероятностный слой, не гарантия). См. [../AUDIT.md#-d7](../AUDIT.md#-d7).
- **🟠 D1 (не здесь):** детектор НЕ ловит path traversal (`detect({"path":"../../etc/passwd"})==False`, доказано). Защита пути — в `tools/filesystem` safe-join, не тут. См. [server.md] (fs_*).
- **🟡 D15:** разделяемый изменяемый `DEFAULT_PATTERNS` (см. Решение 3).

## Что улучшить (линза test-master → набор `tests/virus_injection`)
- Регрессия FP: `"act as a narrator"`, `"disregard the previous take"` НЕ должны блокироваться (после сужения эвристики/word-boundary/allowlist доменных фраз).
- Регрессия FN: если детектор оставляем — base64/пробелы/unicode должны ловиться (нормализация перед матчем); иначе честно пометить детектор как «сигнальный, не барьер».
- Регрессия D15: `add_pattern` на инстансе НЕ меняет `DEFAULT_PATTERNS` (изоляция).
- Онбординг/6мес: без word-boundary и нормализации список паттернов будет расти и давать всё больше FP; новому разработчику неочевидно, что «защита» одновременно дырявая (FN) и мешающая (FP) — зафиксировать это в докстринге файла как явный статус «эвристика-сигнал, не барьер».

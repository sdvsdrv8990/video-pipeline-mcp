## Ось 1 — Границы и дизайн (Bounded Context, антидубль, тонкая обёртка)

- **R-ARCH1 — каждый инструмент в своём `tools/<group>/`, логика в `core/*_core`, обёртка тонкая.**
  *Откуда:* DIM-13/15, F1/F24 (монолит-размещение), F44 (function-local импорты). *Приёмка:* `server.py` = только
  `create_server`/`run_server`/`main`; `tools/` не пуст; хендлеры делегируют в core (уже так — под-1 обмера).
  *GitHub-кандидат:* layout `github/github-mcp-server` (структура групп) — адаптировать, не копировать (§4 `08`).
- **R-ARCH2 — декларативная регистрация: спек-списки → `config/ops/<group>.ops.yaml`, единый источник.**
  *Откуда:* OQ-B2 (спек-списки УЖЕ полу-декларативны), DIM-14. *Приёмка:* `engine.register` читает yaml; НЕТ
  параллельного inline-реестра (yaml ЗАМЕНЯЕТ, не поверх). *GitHub-кандидат:* декларативные tool-манифесты MCP SDK.
- **R-ARCH3 — два хранилища разведены и синхронизированы: `read.json` (данные-SoT) ↔ `.xlsx` (структура/формулы).**
  *Откуда:* B3, G16 (`xlsx_synced=False`). *Приёмка:* построен мост синка read.json→.xlsx; A7-скрипт уникальности
  читает/пишет `read.json`. *GitHub-кандидат:* нет (доменное); openpyxl-паттерны пересчёта.

## Ось 2 — Контракт (`ToolResult`/`ErrorDetail`, коды, parity)

- **R-CONTRACT1 — ЕДИНОЕ ядро реакций: все пути ошибок через `get_error(code, params)`; движки эмитят ТОЛЬКО код.**
  *Откуда:* F43/F5/B2 (реестр обходится хендлерами `_err`; DEFAULT игнорит template), DIM-10. *Приёмка:* strict-xfail
  F43/F5 (`test_audit_fixes`) становятся 🟢; `_err` и `engine._error` идут через реестр; `reaction_class`/`message`/
  `recovery` — из `server_reactions.yaml`, не хардкод в движках. *GitHub-кандидат:* нет (наш реестр); паттерн «errors as data».
- **R-CONTRACT2 — коды всех подсистем ⊂ `server_reactions.yaml`; `KNOWN_ERROR_CODES` синхронен с реестром.**
  *Откуда:* F40 (search-орфаны `QUERY_NOT_FOUND`/`PATH_NOT_FOUND`), DIM-10. *Приёмка:* strict-xfail F40 → 🟢; тест
  паритета «коды кода ⊂ реестр» в CI. *GitHub-кандидат:* нет.
- **R-CONTRACT3 — parity до клиента: `reaction_class`/`recovery` доезжают в MCP `structuredContent`.**
  *Откуда:* D30 (транспорт ГОТОВ — под-7) + F43 (хендлеры не заполняют). *Приёмка:* живой C2 видит class/recovery в error.
  *GitHub-кандидат:* `modelcontextprotocol/conformance` (parity-гейт, DIM-1).

## Ось 3 — Безопасность (ОБА направления) — паттерны/фиксы из `06`, GitHub НЕ ищем

- **R-SEC-IN — containment choke-point держать (G17 ✅), + fail-closed, write-allowlist (F34), no-root (F35).**
  *Откуда:* DIM-3/5/16, `06 §F/§G`. *Приёмка:* `core.paths.safe_resolve` единственный (уже); allowlist default-deny +
  `FILE_TYPE_FORBIDDEN`; bandit-гейт в CI. *Решение:* `06` (не ищем на GitHub).
- **R-SEC-OUT — провенанс workspace-вывода (OUT1/F33), санитизация `_SESSION_LOG` (OUT6/B1 сырой `fact.data`),
  `destructiveHint`+containment на destructive (OUT5), `raw_response`-санитайзер держать (D23/F11 ✅).**
  *Откуда:* DIM-4/16, `06 §B/§D`, B1. *Приёмка:* лог не эхоит сырой `fact.data`; провенанс-маркер на выводе; F11-регрессия 🟢.
  *Решение:* `06`.
- **R-AUTH — app-level OAuth 2.1 + PKCE Resource Server.** *Откуда:* DIM-2, D3/F14 (за туннелем один клиент, IP бесполезен G18).
  *Приёмка:* сервер = Resource Server, per-server scoped token. *Решение:* MCP Python SDK OAuth-хелперы (`06`/M3), обернуть в `core/auth/`.

## Свод зрелости (цель)

Enterprise = **L4 по всем** (или принятый L3-минимум) по `07`. Порядок к L3: **A2 (разблокирует внутреннюю
дисциплину DIM-13/15) → I6 (R-SEC/R-AUTH — красные DIM-2/4/16) ∥ I5 (observability DIM-7) → A6 (R-CONTRACT реакции)
→ A5/A-tables (R-PROD2, search) → I7 (R-TEST2 C2) → P-ось (R-PROD1) → L4 (conformance+quotas+supply-chain)**.
FFmpeg — в самом конце, тогда же в оценку.

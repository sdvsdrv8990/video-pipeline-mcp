# Предложения по улучшению функций (IMP#)

> Здесь предлагаем улучшения **существующих** функций кода (не пробелы — те в `../02_findings.md` как `F#`,
> не новые фичи — те воркстримами в `../01_master_roadmap.md`). Триггер: при чтении спеки видно, что рабочая
> функция могла бы делать лучше/чище/безопаснее, **не меняя контракт**. Принцип владельца: синхронизировать
> не ломая. Каждое предложение проходит через домен-скил перед реализацией.

**Формат:** `IMP#` · целевая функция (файл:символ) · что улучшить · зачем · связь (F#/спека/скил) · статус (💡предложено / 🔨в работе / ✅сделано / ⛔отклонено).

## Как добавлять

Новую строку — с уникальным `IMP#`, ссылкой на конкретный символ кода и на спеку/находку. Реализация:
взять `IMP#` → домен-скил (`reactions-errors`/`mcp-developer`/`anti-hardcode`/…) → правка без смены контракта →
тесты зелёные до/после → статус ✅ + запись решение→факт в `docs/dev/history_*.md`.

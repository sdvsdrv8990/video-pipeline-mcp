# Инструмент: Таблицы (Tables)

## Обзор

Набор MCP-инструментов для работы с табличными данными. Поддерживает CRUD-операции, фильтрацию и пакетную обработку.

## Доступные операции

### `table_list`

Получение списка всех доступных таблиц.

**Возвращает:** массив объектов с именами и метаданными таблиц.

### `table_get`

Получение схемы и данных конкретной таблицы.

**Параметры:**
- `table` (обязательный) — имя таблицы
- `limit` (опциональный) — максимальное количество строк
- `offset` (опциональный) — смещение

**Возвращает:** схему столбцов и данные.

### `table_insert`

Добавление одной или нескольких строк.

**Параметры:**
- `table` (обязательный) — имя таблицы
- `rows` (обязательный) — массив строк для вставки

**Возвращает:** количество добавленных строк.

### `table_update`

Обновление строк по условию.

**Параметры:**
- `table` (обязательный) — имя таблицы
- `where` (обязательный) — условие фильтрации
- `set` (обязательный) — данные для обновления

**Возвращает:** количество обновлённых строк.

### `table_delete`

Удаление строк по условию.

**Параметры:**
- `table` (обязательный) — имя таблицы
- `where` (обязательный) — условие фильтрации

**Возвращает:** количество удалённых строк.

### `table_query`

Выполнение произвольного запроса.

**Параметры:**
- `query` (обязательный) — SQL-подобный запрос
- `params` (опциональный) — параметры запроса

**Возвращает:** результат запроса.

## Доступные таблицы

| Имя таблицы | Описание |
|---|---|
| `video_data` | Основные данные о видео |
| `channel_data` | Информация о каналах |
| `competitor_video_data` | Видео конкурентов |
| `competitor_channel_data` | Каналы конкурентов |
| `network_config` | Конфигурация сетей |
| `niche_network_data` | Данные нишевых сетей |

## Особенности и ограничения

### Безопасность

- Все запросы проходят через валидатор SQL
- **Инъекция заблокирована:** параметризованные запросы, экранирование спецсимволов
- Запрет на DDL-операций (DROP, ALTER, CREATE)
- Чтение только разрешённых таблиц

### Типизация

- Автоматическая приведение типов при вставке/обновлении
- Валидация обязательных полей
- Проверка формата данных (даты, URL, ID)

### Транзакционность

- Операции `insert`, `update`, `delete` выполняются в транзакции
- При ошибке — полный откат
- Пакетные операции атомарны

### Кеширование

- Результаты `get` кешируются с TTL 60 сек
- Запись инвалидирует кеш таблицы
- Схема таблицы кешируется дольше (10 минут)

### Ошибки

- `400` — невалидный запрос / нарушение схемы
- `404` — таблица не найдена
- `409` — конфликт уникальности
- `422` — нарушение внешнего ключа

## Примеры использования

```json
// Список таблиц
{}

// Получение данных видео
{"table": "video_data", "limit": 10}

// Вставка новой записи
{
  "table": "video_data",
  "rows": [{"video_id": "abc123", "title": "Новое видео", "status": "draft"}]
}

// Обновление статуса
{
  "table": "video_data",
  "where": {"video_id": "abc123"},
  "set": {"status": "published"}
}
```

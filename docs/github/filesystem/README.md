# Инструмент: Файловая система (Filesystem)

## Обзор

Набор MCP-инструментов для работы с файловой системой хоста. Инструменты предоставляют безопасный доступ к файлам с контролем путей и валидацией.

## Доступные операции

### `fs_list`

Получение списка файлов и директорий по указанному пути.

**Параметры:**
- `path` (обязательный) — путь к директории

**Возвращает:** массив записей с именами, типами (file/dir) и размерами.

### `fs_read`

Чтение содержимого файла.

**Параметры:**
- `path` (обязательный) — путь к файлу
- `offset` (опциональный) — смещение для чтения
- `limit` (опциональный) — максимальный размер

**Возвращает:** содержимое файла в виде строки.

### `fs_write`

Запись данных в файл. Создаёт файл, если он не существует.

**Параметры:**
- `path` (обязательный) — путь к файлу
- `content` (обязательный) — содержимое

**Возвращает:** подтверждение записи.

### `fs_delete`

Удаление файла.

**Параметры:**
- `path` (обязательный) — путь к файлу

**Возвращает:** подтверждение удаления.

## Особенности и ограничения

### Безопасность путей

- Все пути разрешаются относительно базовой директории workspace
- **Path traversal заблокирован:** попытки использовать `..` для выхода за пределы workspace отклоняются
- Симлинки не следуют — работаем только с реальными путями

### Валидация

- Проверка расширений файлов (ограничения на опасные типы)
- Проверка размера файлов при записи
- Запрет на перезапись критических файлов конфигурации

### Кеширование

- Списки директорий кешируются на короткий TTL
- Изменения файлов инвалидируют кеш

### Ошибки

- `404` — файл не найден
- `403` — нет доступа к пути
- `413` — файл слишком большой
- `422` — невалидный путь

## Примеры использования

```json
// Список файлов
{"path": "/workspace/videos"}

// Чтение файла
{"path": "/workspace/scripts/ffmpeg_filter.txt"}

// Запись нового файла
{"path": "/workspace/output/result.txt", "content": "Готово"}
```
